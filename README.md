# OCR Quality Validation (Chandra)

A Streamlit app that **statistically validates page-wise [Chandra](https://github.com/) OCR
output** and surfaces the pages a human must check — then exports a faulty-pages review
package. It is tuned to the **real Chandra output schema** (nested JSON block tree, HTML
content, `inference_failed` flag) and **does not assume per-token confidence** (Chandra
does not emit it).

The pipeline converges two OCR sources onto one identical compute + report path:

```
Ingest → Processing → Faulty Pages → Verdict → Export
```

---

## Quick start (one command)

```bash
pip install -r ocr_qa/requirements.txt
streamlit run ocr_qa/app.py
```

Then in the **Ingest** tab:
1. Upload a source document (PDF / DOCX / PPTX / ZIP-of-images).
2. Choose **OCR status**:
   - **OCR already done** → upload `output.json`, `output.md`, `output.metadata.json`
     (the three Chandra artefacts), and **optionally `output_chunks.json`**. *Fully functional.*
   - **Run Chandra OCR** → stub transport in this build (the **parse** path is real; wire
     `RealChandraClient` to your Chandra endpoint to enable it).
3. Click **Analyze**.

> Try it immediately with the bundled synthetic fixtures in `ocr_qa/sample/good_page/`
> (passes) and `ocr_qa/sample/bad_page/` (fails) — upload each triple via "OCR already done".

## What it computes — 14 statistical metrics

Each metric returns a deterministic, templated card: *value · what it measures · how
computed · why this score (numbers + threshold) · evidence from this page*. Justifications
are **never** generated by an LLM.

| Group | Metric | Key | Signal |
|------|--------|-----|--------|
| Engine & structure | Inference-Failure Rate | `IFR` | Chandra's own `inference_failed` flag (**primary signal**) |
| | Block-Count & Source Consistency | `BCC` | json blocks vs metadata `num_blocks`, vs doc median |
| | Block-Geometry Coherence | `BGC` | degenerate / out-of-bounds / overlapping / order-inverted boxes |
| Markup | HTML/Markup Well-formedness | `HMW` | tag balance, broken entities |
| | Table Structure Integrity | `TSI` | header, ragged rows, empty cells, stray colspans |
| Lexical | Lexical Validity Ratio | `LVR` | real-word rate (wordfreq + spellchecker + whitelist) |
| | OOV Pronounceability | `OOV` | gibberish vs plausible brand words |
| | Stopword Frequency Conformance | `SFC` | function-word ratio vs language band |
| Statistical | Character Entropy & Mojibake | `CED` | KL vs language baseline + explicit mojibake |
| | N-gram Perplexity / Fluency | `PPL` | char-trigram surprisal, self-calibrated on the doc |
| | Lexical Diversity / TTR | `TTR` | looping vs fragmentation |
| Surface | Word-Shape / Tokenization | `WSA` | merged / over-segmented / hyphen-break / l↔1,O↔0 |
| | Punctuation & Sentence | `PSW` | unmatched brackets/quotes, sentence sanity |
| Agreement | Repetition & Cross-Source | `XSA` | repetition loop + JSON↔MD + native-text CER/WER |

**Optional 4th artefact — `output_chunks.json`:** when supplied, the chunk-level
`content` and `table_metrics` feed three existing metrics (no 15th metric is added):
`XSA` gains a JSON↔chunk text-agreement sub-part, `TSI` cross-checks parsed tables against
the chunk's `has_table` / `table_count` (catches dropped tables), and `BCC` flags any page
absent from all chunks (content drop). Chunk `page_ids` are 0-based and may span several
pages — the parser detects the offset and attributes signals only when unambiguous.

**Multilingual & domain aware:** accented Latin letters are valid; pharma/brand proper
nouns (Nemluvio, Galderma, Gemeinsamer…) pass via a whitelist + pronounceability gate.
**Tables/figures are not scored as prose** — they have their own structural metrics.

### Scoring & verdict

- **PQS** (page) = weighted sum of available metric scores; weights renormalise over the
  metrics that applied.
- **DQS** (document) = token-weighted mean of PQS (small pages can't dominate).
- A page is **faulty** if `PQS < T_page` **or** any **HARD** condition fires:
  `inference_failed` on a content block · mojibake above bound · repetition loop ·
  `LVR < 0.80` · native CER `> 0.25` · block-count mismatch beyond tolerance.
- **DOCUMENT VERDICT = PASSED** iff `DQS ≥ T_doc` **and** faulty ratio `≤ doc_flag_ratio`
  **and** no page hit a HARD condition. Otherwise **FAILED**, with a numeric reason and the
  exact named rule(s) that caused it. *Deterministic: same inputs → same verdict & wording.*

All thresholds and weights are editable live in the **sidebar** (Sec 9 defaults pre-filled).

## Dependencies & optional components

### LibreOffice (DOCX / PPTX only)
DOCX/PPTX are converted to PDF via **LibreOffice headless** (`soffice`). PDF and
ZIP-of-images need no external binary.

- Install LibreOffice and ensure `soffice` is on `PATH` (the app also probes the standard
  Windows/macOS/Linux install locations).
- If it is missing, the app **degrades with a clear message** instead of crashing — export
  the document to PDF yourself and upload that.

### Perplexity backend
`PPL` is pluggable via the sidebar:
- **`ngram`** (default) — a lightweight char-trigram model trained on the document itself.
  **No model download.** Recommended for multilingual/domain text.
- **`kenlm`** / **`gpt2`** — placeholders; they fall back to `ngram` with a note unless you
  add the optional deps (`kenlm`, or `transformers`+`torch`) and wire the backend.

> **Windows note:** PyMuPDF must be importable as `pymupdf` (the legacy `fitz` alias can be
> shadowed). The app handles this automatically. If PDF rendering fails, run
> `pip install --upgrade PyMuPDF`. Set `PYTHONIOENCODING=utf-8` if your console errors on
> mojibake glyphs.

## Human-review export (faulty pages only)

The **Export** tab builds one zip:
- `review.json` — per faulty page: PQS, every metric (incl. evidence), problems, image name
- `review.csv` — page, PQS, top-3 problems
- `review.md` — one section per faulty page (image ref + faults + justifications)
- `pages/` — the faulty page images

`send_for_review(package)` is a marked stub for a future webhook / Label-Studio push (no
external integration runs in this build).

## Tests

```bash
pip install -r ocr_qa/requirements.txt
pytest ocr_qa/tests -q
```

Covers: schema parsing (nested tree, HTML strip, MD split, metadata, defensive); each metric
clean-vs-garbled; HARD conditions; PASS/FAIL verdict + determinism; ingestion of all source
types; the export package; and a Streamlit `AppTest` smoke run of every stage.
